✏ Pencil Production Line

Manufacturing Simulation with Cloud Monitoring — SRH Advanced Programming

What is it?

A real-time manufacturing simulation of a 4-station wooden pencil assembly line. The app runs locally in Python, while all production metrics are streamed live to InfluxDB Cloud and visualised on Grafana Cloud — no local database or dashboard installation needed.

The Production Line

Each pencil travels through four sequential stations — Core Insertion, Body Bonding, Ferrule Crimping, and Eraser Insertion. Every station monitors a key physical parameter (insertion force, cure temperature, crimp pressure, insertion depth) and raises a specific, human-readable defect reason if a measurement falls out of tolerance. A pencil that fails any station is immediately routed to the reject bin with a full diagnosis.

Tech Stack

The HMI is built with FastAPI + WebSockets for real-time push updates, a plain HTML/CSS/JS operator screen with Start / Stop / Reset controls and a live fault banner, and the influxdb-client library for batched cloud writes. The Grafana dashboard (temperature, machine state, good vs. defective parts, defect Pareto) is provisioned programmatically via the Grafana HTTP API — reproducible from a single script.

Key Features

  • Explainable defects — every reject carries a station name, reason code, and measured value
  • Overheat fault at Body Bonding → machine FAULTED, HMI shows red alarm, Reset required
  • CMMS module tracks cycles since maintenance and raises maintenance-due alerts
  • Cloud resilience — write failures are logged and surfaced in the HMI footer; the simulation never crashes on network errors
  • Unit tests with no .env requirement — engine is fully decoupled from cloud dependencies
